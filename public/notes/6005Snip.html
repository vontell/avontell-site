Finite Automata

A finite automaton is a machine with a finite number of gates that reads one input character at a time. With each bit inputted, the DFA (deterministic finite automata) moves to a new state based on that current input. The goal of the FA is to check some property of an input string. Finite automatons can be represented in the following way:

Example of a finite automaton

$$Tuple\ of\ values\ : (Q, \Sigma, \delta, q_0, F)$$

$$ \begin{equation*} \begin{split} Q & = Possible\ States \\ \Sigma & = Alphabet \\ \delta &= Transition\ Function\ (Q\times \Sigma \rightarrow Q) \\ q_0 &= Start\ State\ (q_0 \in Q) \\ F &= Accept\ States\ (F \subseteq Q) \\ \end{split} \end{equation*} $$

Languages

A DFA D accepts a language (set of finite strings) \(L \subseteq \Sigma^*\). A commons choice for a language is the boolean alphabet, or simply \(\Sigma = \{0,1\}\). If that is our language, than \(\Sigma^* = \{\oslash, 0, 1, 00, 01, 10, 11, 000, ... \}\) is our set of words, in lexicographic order (and where each word is finite).

A DFA D accepts a language if \(\forall x \in \Sigma^*, D\ accepts\ x \implies x \in L\)

Additionally, a language \(L\) is regular if there's some DFA that accepts it. In other words, the language \(L\) can be recognized by some finite automaton.

If the language we are working with is finite, then we can make a huge tree where it's depth is equal to the longest word in the language. With this tree we can cover all possible combination of letters, while putting accepting states at the points where words terminate, and reject states past the end of the tree.

However, are there any languages that are not regular? There are!

Theorem: \(L \subseteq \{L, R\}^*\) with some number of \(L\) and \(R\) is not regular. In other words, there is no finite automaton that can act as a palindrome detector. We can prove this using the pigeonhole principle.

Let's start by looking at the state of automaton at the center of the string. We can first realize that, while moving through the first half of the string, we have encountered a finite number of states (the number of states in our automaton), but there are an infinite number of ways we could have visited those states. Through the pigeonhole principle, this means that there must be multiple ways to get to the same state within the first half of the string.

Now, let's assume we have two strings \(x\) and \(y\), chosen such that the state reached halfway through the string is the same. If we take the first half of \(x\) and the second half of \(y\) and "glue" them together, then the automaton will accept even though the two halves of the string are different, because of that similar middle state.

Pumping Lemma

If \(A \subseteq \Sigma^*\) is regular, \(\exists p\) such that if \(S \in A\), \(|S| \ge p\), then \(S=xyz\) with \(|y| \gt 0\) such that \(xy^iz\ \in A\ \forall i \ge 0\)

Nondeterministic Finite Automata (NDFA)

An NDFA works just like a DFA, but an input is allowed to change the state of the machine to more than one state. For example, with an input of 1, the NDFA might go from state A to B, or A to C. Therefore, \(\delta\) has a new definition:$$\delta:(Q \times(\Sigma \cup \{\epsilon\}) \rightarrow 2^Q$$

An NDFA \(N\) accepts if there exists a path compatible with \(x\) that ends in an accept state. Null transitions are also allowed, or transitions you can take without reading any symbols. Finally, there is an important theorem: "NDFA"="DFA", and every language recognized by an NDFA is regular.
