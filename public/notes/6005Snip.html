What is a circuit?

In this class, we will not be talking about electrical circuits, which have closed cycles. Rather, for computer science, our circuits cannot have any cycles. We have an input, some logic it passes through, and some output. For example, boolean functions take in an n-bit string and output a single bit answer:$$f: \{0,1\}^n \rightarrow \{0,1\}$$

Some examples of boolean functions include OR, parity, and the majority function.

The majority circuit.

Now, how can we interpret the input? We do so by using logic gates to decompose our input into usable chunks. For example, we may use the following gates:

Gate Equation Representation
NOT \(x+1\ (mod\ 2)\)
AND \(xy\)
OR \(x+y-xy\)
XOR \(x+y\ (mod\ 2)\)

The overall idea is to apply these logic operations on inputs, and then use those outputs in other logical operations. Remember that these logic gates are also associative, meaning they can be cascaded and stringed together to form operations on large input strings.

Universal gates

However, how many and which gates can be grouped together in order to be considered universal; that is, by composing anything out of this designated set of gates, we can represent every boolean function \(f: \{0,1\}^n \rightarrow \{0,1\}\) (where 0 and 1 are also allowed).

Some examples of non-universal gates includes \(\{AND\}\), \(\{OR\}\), \(\{NOT\}\), \(\{\}\), \(\{AND, OR\}\), and \(\{XOR, NOT\}\). Why are these not universal? We can prove this by using a preserved invariant, looking at the equation representations of the above gates.

For example, let's take a look at the set \(\{AND, OR\}\). Turns out that this set of gates can only give rise to monotone boolean functions. This means that if \(x \le y\), then \(f(x) \le f(y)\). On other words, changing an input from 0 to 1 will cause output to change from a 0 to 1. The output only ever increases as the input increases, which makes operations such as NOT impossible, as it requires a decrease in output as input is increased. Only monotone functions can be created from the set \(\{AND, OR\}\) (this also means that every monotone function can be created with this set).

The set \(\{XOR, NOT\}\) is also not linear due to the degree of its equations. Both gates are linear, but AND and OR require degree two polynomials, making it impossible to derive them.

From these two examples, it was actually found that if a the set of gates are only monotone and / or linear, then they will not be universal. In fact, it is interesting that a set being universal is quite common; it is the non-universal sets that are rare.

Review of asymptotic notation

The following is a list of important definitions of asymptotic bounds:

  • \(f(n) = O(g(n))\) if \(\exists A > 0 \ s.t.\ f(n) \le Ag(n)\) for all sufficiently large \(n\)
  • \(f(n) = \Omega(g(n))\) if \(g(n)=O(f(n))\)
  • \(f(n) = \Theta(g(n))\) if \(f(n)=O(g(n))\ and\ \Omega(g(n))\)
  • \(f(n) = o(g(n))\) if \(\forall c \ge 0, f(n) \lt cg(n)\) for all sufficiently large \(n\)

Theorem: {AND, OR, NOT} is universal.

We can prove this theorem through an example, and also through a formal proof.

The number of gates needed for an operation on an n-bit string is \(O(n2^n)\). It turns actually turns out that \(\{AND, NOT\}\) is universal, since we can make an OR gate use De Morgan's law. Also, this means that \(\{NAND\}\) by itself is universal.

But using just NAND gates, can we say the upperbound on how many gates are needed to be represent any boolean function?

Claude Shannon provided some insight on this in 1949, when he discovered that "There necessarily exists a boolean function on n - bits such that any circuit requires \(\Omega(2^n/n)\) gates. In fact, almost any function does."

This can be proved through a counting argument - there are \(2^{2^n}\) boolean functions, since the truth table has size \(2^n\), and each output is independent. By using just NAND gates, with an n-bit input, 0, 1, and where each NAND takes two bits, we can find the number of possible circuits we can make. Since for every NAND we add, there is a new input, we have that we can make \({n+T+2 \choose 2}\) different circuits.

With that answer and some extra math, we find that \(T = \Omega(2^n/n)\), confirming the lower bound on the number of gates needed to create every boolean function

Circuit Limitations

Circuits are not too shabby at being a computational model; however, it does have its limits. They can only handle inputs of fixed size, and they also have a lack of memory. We want to handle input strings of any size, but circuits are non-uniform. A uniform model will take in any size input, which is achievable by finite automata, which we will see next time.
